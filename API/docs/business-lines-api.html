Business Lines Endpoints

Manage organizational structure and business hierarchies

GET /business_lines Get all business lines with portfolio and product line mapping

Description

Retrieves the complete organizational structure showing the relationship between strategic portfolios and product lines. Results are ordered by strategic portfolio.

Response Fields

Field Type Description
strategic_portfolio string High-level strategic business portfolio
product_line string Specific product line within the portfolio

Response Example

[
  {
    "strategic_portfolio": "Auto Insights",
    "product_line": "Auto Claims"
  },
  {
    "strategic_portfolio": "Auto Insights",
    "product_line": "Auto Underwriting"
  },
  {
    "strategic_portfolio": "Market & Sell",
    "product_line": "Commercial"
  },
  {
    "strategic_portfolio": "Market & Sell",
    "product_line": "Personal Lines"
  },
  {
    "strategic_portfolio": "Commercial Operations",
    "product_line": "Small Business"
  }
]

PowerShell Example

Invoke-WebRequest -Uri "http://localhost:5000/business_lines" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}

GET /product_lines/{strategic_portfolio} Get product lines for a specific strategic portfolio

Description

Retrieves all product lines that belong to a specific strategic portfolio, enabling hierarchical navigation and filtering.

Path Parameters

Name Type Description Required
strategic_portfolio string Strategic portfolio name (URL encoded) Yes

Response Example

[
  {
    "strategic_portfolio": "Market & Sell",
    "product_line": "Commercial"
  },
  {
    "strategic_portfolio": "Market & Sell",
    "product_line": "Personal Lines"
  }
]

PowerShell Example

# URL encode the portfolio name
$portfolio = [System.Web.HttpUtility]::UrlEncode("Market & Sell")
Invoke-WebRequest -Uri "http://localhost:5000/product_lines/$portfolio" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}

cURL Example

curl -X GET "http://localhost:5000/product_lines/Market%20%26%20Sell" \
  -H "Content-Type: application/json"

Dynamic Filtering

Create cascading dropdown interfaces

# Step 1: Get all portfolios for dropdown
$portfolios = (Invoke-WebRequest -Uri "http://localhost:5000/strategic_portfolios" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}).Content | ConvertFrom-Json

# Step 2: User selects "Market & Sell"
$selectedPortfolio = "Market & Sell"
$encodedPortfolio = [System.Web.HttpUtility]::UrlEncode($selectedPortfolio)

# Step 3: Get product lines for selected portfolio
$productLines = (Invoke-WebRequest -Uri "http://localhost:5000/product_lines/$encodedPortfolio" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}).Content | ConvertFrom-Json

Business Analysis

Analyze portfolio distribution and coverage

# Analyze business line distribution
$businessLines = (Invoke-WebRequest -Uri "http://localhost:5000/business_lines" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}).Content | ConvertFrom-Json

# Count product lines per portfolio
$portfolioStats = $businessLines | Group-Object strategic_portfolio | 
    Select-Object Name, Count | 
    Sort-Object Count -Descending

Write-Host "Portfolio Distribution:" -ForegroundColor Green
$portfolioStats | Format-Table -AutoSize

Integration with Other APIs

How Business Lines API connects with the broader PMO ecosystem

Projects API

Projects are aligned with business lines - use this for project categorization and filtering.

# Get business lines for project filtering
$portfolios = (Invoke-WebRequest -Uri "http://localhost:5000/strategic_portfolios").Content | ConvertFrom-Json

# Filter projects by business alignment
$projects = (Invoke-WebRequest -Uri "http://localhost:5000/projects").Content | ConvertFrom-Json
$marketSellProjects = $projects | Where-Object { $_.strategic_portfolio -eq "Market & Sell" }

Resources & Managers

Resources and managers are assigned to strategic portfolios - create organizational views.

# Cross-reference resources with business structure
$businessLines = (Invoke-WebRequest -Uri "http://localhost:5000/business_lines").Content | ConvertFrom-Json
$resources = (Invoke-WebRequest -Uri "http://localhost:5000/resources").Content | ConvertFrom-Json
$managers = (Invoke-WebRequest -Uri "http://localhost:5000/managers").Content | ConvertFrom-Json

# Build organizational chart
$orgChart = $businessLines | Group-Object strategic_portfolio | ForEach-Object {
    @{
        Portfolio = $_.Name
        ProductLines = $_.Group.product_line
        Resources = ($resources | Where-Object { $_.strategic_portfolio -eq $_.Name }).Count
        Managers = ($managers | Where-Object { $_.strategic_portfolio -eq $_.Name }).manager_name
    }
}

Screener API

Use business line data to build dynamic filtering criteria for advanced queries.

# Build dynamic screener filters
$portfolios = (Invoke-WebRequest -Uri "http://localhost:5000/strategic_portfolios").Content | ConvertFrom-Json

# Create filter for specific portfolio projects
$filterBody = @{
    filters = @(
        @{
            column = "strategic_portfolio"
            operator = "="
            value = "Market & Sell"
        }
    )
    logical_operator = "AND"
    fields = @("project_id", "project_name", "strategic_portfolio", "product_line")
} | ConvertTo-Json

Invoke-WebRequest -Uri "http://localhost:5000/screener_projects" -Method POST -Headers @{"Content-Type"="application/json"} -Body $filterBody

Best Practices

Tips for effective business line management

URL Encoding

Always URL encode strategic portfolio names when using them in path parameters, especially those containing special characters like "&".

# Correct way to handle special characters
$portfolio = "Market & Sell"
$encoded = [System.Web.HttpUtility]::UrlEncode($portfolio)
# Results in: "Market%20%26%20Sell"

Hierarchical Navigation

Use the three-tier approach: start with strategic portfolios, then drill down to specific product lines for optimal user experience.

Caching Strategy

Business line data changes infrequently - consider caching responses for better performance in client applications.
