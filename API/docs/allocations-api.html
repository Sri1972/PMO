Allocation Endpoints

Complete resource allocation management and tracking

GET /allocations Get all resource allocations with calculated hours and costs

Description

Retrieves all resource allocations with enriched data including planned hours, costs, and time-off adjustments.

Response Fields

Field Type Description
allocation_id integer Unique allocation identifier
project_id integer Associated project ID
resource_id integer Associated resource ID
allocation_start_date string Start date (YYYY-MM-DD)
allocation_end_date string End date (YYYY-MM-DD)
allocation_pct number Allocation percentage
allocation_hrs_per_week number Hours per week allocation
resource_hours_planned number Calculated planned hours
resource_cost_planned number Calculated planned cost

PowerShell Example

Invoke-WebRequest -Uri "http://localhost:5000/allocations" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}
GET /allocations/project Get allocations for specific project(s) with planned vs actual tracking

Query Parameters

Name Type Description Required
project_ids array[integer] List of project IDs to filter by Yes

Additional Response Fields

  • resource_hours_actual: Actual hours worked from timesheets
  • resource_cost_actual: Actual cost based on timesheet data
  • timesheet_start_date: First timesheet entry date
  • timesheet_end_date: Last timesheet entry date

PowerShell Example

Invoke-WebRequest -Uri "http://localhost:5000/allocations/project?project_ids=1&project_ids=5&project_ids=10" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}

POST /allocate Create or update resource allocations (supports bulk operations)

Request Body

Accepts an array of allocation objects for bulk operations.

[
  {
    "allocation_id": 101,
    "project_id": 1,
    "resource_id": 25,
    "allocation_start_date": "2025-01-01",
    "allocation_end_date": "2025-06-30",
    "allocation_pct": 50,
    "allocation_hrs_per_week": null
  },
  {
    "allocation_id": 102,
    "project_id": 1,
    "resource_id": 30,
    "allocation_start_date": "2025-02-01",
    "allocation_end_date": "2025-08-31",
    "allocation_pct": null,
    "allocation_hrs_per_week": 20
  }
]

Required Fields

Field Type Description
allocation_id integer Unique allocation identifier
project_id integer Project to allocate to
resource_id integer Resource being allocated
allocation_start_date string Start date (YYYY-MM-DD)
allocation_end_date string End date (YYYY-MM-DD)
Note: Either allocation_pct OR allocation_hrs_per_week must be provided (not both).

PowerShell Example

$body = @(
    @{
        allocation_id = 101
        project_id = 1
        resource_id = 25
        allocation_start_date = "2025-01-01"
        allocation_end_date = "2025-06-30"
        allocation_pct = 50
        allocation_hrs_per_week = $null
    }
) | ConvertTo-Json

Invoke-WebRequest -Uri "http://localhost:5000/allocate" `
  -Method POST `
  -Headers @{"Content-Type"="application/json"} `
  -Body $body

Common Use Cases

Real-world examples for allocation management

Project Progress Tracking

Get planned vs actual progress for a project

# Get detailed allocations with actual tracking
Invoke-WebRequest -Uri "http://localhost:5000/allocations/project?project_ids=1" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}

Role-Based Analysis

Analyze resource allocation by role

# Get role summaries for multiple projects
Invoke-WebRequest -Uri "http://localhost:5000/allocations/resource_role_summary?project_ids=1&project_ids=5&project_ids=10" `
  -Method GET `
  -Headers @{"Content-Type"="application/json"}

Bulk Allocation Creation

Create multiple allocations at once

# Bulk allocation for project team
$allocations = @(
    @{ allocation_id=101; project_id=1; resource_id=25; allocation_start_date="2025-01-01"; allocation_end_date="2025-06-30"; allocation_pct=100 },
    @{ allocation_id=102; project_id=1; resource_id=30; allocation_start_date="2025-01-01"; allocation_end_date="2025-06-30"; allocation_pct=50 }
) | ConvertTo-Json

Invoke-WebRequest -Uri "http://localhost:5000/allocate" -Method POST -Headers @{"Content-Type"="application/json"} -Body $allocations
