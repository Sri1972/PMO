CRUD Operations

Core project management operations for creating, reading, updating project data

GET /projects
Retrieve all projects with resource allocation summaries

Fetches all projects from the PMO system with comprehensive details including resource allocation summaries, costs, and role breakdowns. This endpoint automatically enriches project data with calculated fields.

Key Response Fields

Field Type Description
project_id integer Unique project identifier
project_name string Project name/title
strategic_portfolio string Strategic business portfolio
current_status string Current project status
start_date_est date Estimated start date (YYYY-MM-DD)
end_date_est date Estimated end date (YYYY-MM-DD)
project_resource_hours_planned number Total planned resource hours
project_resource_cost_planned number Total planned resource cost
project_resource_hours_actual number Total actual resource hours
resource_role_summary object Breakdown by resource roles

PowerShell Example

# Get all projects with resource summaries
$response = Invoke-WebRequest -Uri "http://localhost:5000/projects" -Method GET
$projects = $response.Content | ConvertFrom-Json

# Display projects summary
Write-Host "Total Projects: $($projects.Count)"
Write-Host ""

# Show project overview
$projects | Select-Object project_name, strategic_portfolio, current_status, project_resource_hours_planned, project_resource_cost_planned | Format-Table

# Show detailed project information
foreach ($project in $projects) {
    Write-Host "Project: $($project.project_name)"
    Write-Host "  Portfolio: $($project.strategic_portfolio)"
    Write-Host "  Status: $($project.current_status)"
    Write-Host "  Planned Hours: $($project.project_resource_hours_planned)"
    Write-Host "  Planned Cost: `$$($project.project_resource_cost_planned)"
    
    if ($project.resource_role_summary -and $project.resource_role_summary.PSObject.Properties.Count -gt 0) {
        Write-Host "  Role Summary:"
        $project.resource_role_summary.PSObject.Properties | ForEach-Object {
            Write-Host "    $($_.Name): $($_.Value.total_resource_hours_planned) hours"
        }
    }
    Write-Host ""
}

GET /projects/{project_id}
Retrieve a specific project by ID

Fetches detailed information for a single project including all resource allocation summaries and role breakdowns.

Path Parameters

Parameter Type Required Description
project_id integer Yes Unique identifier for the project

PowerShell Example

# Get specific project details
$projectId = 202
$response = Invoke-WebRequest -Uri "http://localhost:5000/projects/$projectId" -Method GET
$project = ($response.Content | ConvertFrom-Json)[0]  # API returns array with single item

# Display comprehensive project details
Write-Host "=== PROJECT DETAILS ==="
Write-Host "Name: $($project.project_name)"
Write-Host "ID: $($project.project_id)"
Write-Host "Portfolio: $($project.strategic_portfolio)"
Write-Host "Product Line: $($project.product_line)"
Write-Host "Status: $($project.current_status) ($($project.rag_status))"
Write-Host "Type: $($project.project_type)"
Write-Host ""
Write-Host "=== TIMELINE ==="
Write-Host "Estimated: $($project.start_date_est) to $($project.end_date_est)"
if ($project.start_date_actual) {
    Write-Host "Actual Start: $($project.start_date_actual)"
}
Write-Host ""
Write-Host "=== RESOURCE ALLOCATION ==="
Write-Host "Planned Hours: $($project.project_resource_hours_planned)"
Write-Host "Planned Cost: `$$($project.project_resource_cost_planned)"
Write-Host "Actual Hours: $($project.project_resource_hours_actual)"
Write-Host "Actual Cost: `$$($project.project_resource_cost_actual)"
Write-Host ""
Write-Host "=== FINANCIALS ==="
Write-Host "Revenue Estimate (Current Year): `$$($project.revenue_est_current_year)"
Write-Host "Vitality: $($project.vitality)"
Write-Host "Strategic Importance: $($project.strategic)"
Write-Host ""
Write-Host "Description: $($project.project_description)"
POST /projects
Add new project or update existing (upsert)

Creates a new project or updates an existing one based on the combination of project_name, strategic_portfolio, and product_line. This endpoint supports comprehensive project data including financial estimates and timeline information.

Request Body Schema (Key Fields)

Field Type Required Description
project_name string Yes Project name (part of unique key)
strategic_portfolio string Yes Strategic portfolio (part of unique key)
product_line string Yes Product line (part of unique key)
project_type string No Type of project (Enhancement, New, etc.)
current_status string No Current project status
start_date_est date No Estimated start date (YYYY-MM-DD)
end_date_est date No Estimated end date (YYYY-MM-DD)
revenue_est_current_year number No Revenue estimate for current year

PowerShell Example

# Create comprehensive project
$body = @{
    project_name = "Mobile App Development"
    strategic_portfolio = "Technology"
    product_line = "Mobile Applications"
    project_type = "New Development"
    project_description = "Develop new mobile application for customer engagement"
    current_status = "Planning"
    rag_status = "Green"
    vitality = "High"
    strategic = "Core"
    aim = "Increase customer engagement through mobile platform"
    start_date_est = "2024-03-01"
    end_date_est = "2024-12-31"
    revenue_est_current_year = 150000
    revenue_est_current_year_plus_1 = 300000
    revenue_est_current_year_plus_2 = 450000
    technology_project = $true
    added_by = "Project Manager"
    added_date = "2024-01-15"
    comments = "High priority project for Q2 launch"
} | ConvertTo-Json

$response = Invoke-WebRequest -Uri "http://localhost:5000/projects" -Method POST -Body $body -ContentType "application/json"
$result = $response.Content | ConvertFrom-Json

Write-Host "Project Status: $($result.message)"

# Verify project was created
$checkResponse = Invoke-WebRequest -Uri "http://localhost:5000/projects" -Method GET
$projects = $checkResponse.Content | ConvertFrom-Json
$newProject = $projects | Where-Object { $_.project_name -eq "Mobile App Development" }

if ($newProject) {
    Write-Host "Project successfully created with ID: $($newProject.project_id)"
}

Timeline Management

Endpoints for managing project milestones and timeline sequences

GET /projects/timelines/{project_id}
Get project timeline milestones

Retrieves all timeline milestones for a project, ordered by sequence number.

Path Parameters

Parameter Type Required Description
project_id integer Yes Project identifier

PowerShell Example

# Get project timeline
$projectId = 202
$response = Invoke-WebRequest -Uri "http://localhost:5000/projects/timelines/$projectId" -Method GET
$timelines = $response.Content | ConvertFrom-Json

Write-Host "Project Timeline Milestones:"
Write-Host "="*50

foreach ($milestone in $timelines) {
    Write-Host "[$($milestone.sequence)] $($milestone.milestone)"
    Write-Host "  Description: $($milestone.milestone_description)"
    Write-Host "  Period: $($milestone.start_date) to $($milestone.end_date)"
    Write-Host ""
}

POST /projects/timelines/{project_id}
Add or update project timeline milestone

Creates a new milestone or updates an existing one. If sequence is not provided, it will be auto-assigned as the next available number.

Request Body Schema

Field Type Required Description
milestone string Yes Milestone name (unique per project)
milestone_description string No Detailed milestone description
start_date date No Milestone start date (YYYY-MM-DD)
end_date date No Milestone end date (YYYY-MM-DD)
sequence integer No Order sequence (auto-assigned if omitted)

PowerShell Example

# Add new milestone
$projectId = 202
$body = @{
    milestone = "Development Phase"
    milestone_description = "Core application development and feature implementation"
    start_date = "2024-03-11"
    end_date = "2024-05-31"
    sequence = 4
} | ConvertTo-Json

$response = Invoke-WebRequest -Uri "http://localhost:5000/projects/timelines/$projectId" -Method POST -Body $body -ContentType "application/json"
$result = $response.Content | ConvertFrom-Json

Write-Host "Timeline Status: $($result.message)"
POST /projects/timelines/reorder/{project_id}
Reorder timeline milestones

Updates the sequence order of multiple milestones in a single operation.

PowerShell Example

# Reorder timeline milestones
$projectId = 202
$body = @(
    @{ milestone = "Project Kickoff"; sequence = 1 },
    @{ milestone = "Design Phase"; sequence = 2 },
    @{ milestone = "Requirements Gathering"; sequence = 3 },
    @{ milestone = "Development Phase"; sequence = 4 }
) | ConvertTo-Json

$response = Invoke-WebRequest -Uri "http://localhost:5000/projects/timelines/reorder/$projectId" -Method POST -Body $body -ContentType "application/json"
$result = $response.Content | ConvertFrom-Json

Write-Host "Reorder Status: $($result.message)"
DELETE /projects/timelines/{project_id}
Delete a timeline milestone

Removes a specific milestone from the project timeline.

Request Body Schema

Field Type Required Description
milestone string Yes Name of milestone to delete

PowerShell Example

# Delete milestone
$projectId = 202
$body = @{
    milestone = "Old Milestone"
} | ConvertTo-Json

$response = Invoke-WebRequest -Uri "http://localhost:5000/projects/timelines/$projectId" -Method DELETE -Body $body -ContentType "application/json"
$result = $response.Content | ConvertFrom-Json

Write-Host "Deletion Status: $($result.message)"

Filtering & Search

Advanced filtering and search capabilities for project data

GET /projects_filter
Filter projects by portfolio, product line, or technology flag

Simple filtering endpoint for common project categorization fields.

Query Parameters

Parameter Type Required Description
strategic_portfolio string No Filter by strategic portfolio
product_line string No Filter by product line
technology_project string No Filter by technology flag (true/false)

PowerShell Example

# Filter technology projects
$params = @{
    strategic_portfolio = "Technology"
    technology_project = "true"
}

$queryString = ($params.GetEnumerator() | ForEach-Object { "$($_.Key)=$($_.Value)" }) -join "&"
$uri = "http://localhost:5000/projects_filter?$queryString"

$response = Invoke-WebRequest -Uri $uri -Method GET
$projects = $response.Content | ConvertFrom-Json

Write-Host "Technology Projects in Technology Portfolio:"
$projects | Select-Object project_name, current_status, project_resource_hours_planned | Format-Table
POST /projects/dynamic_filter
Advanced dynamic filtering with custom fields

Powerful filtering endpoint that allows complex queries with multiple conditions, logical operators, and field selection. Perfect for custom dashboards and reports.

Request Body Schema

Field Type Required Description
filters array No Array of filter objects
logical_operator string No "AND" or "OR" (default: "AND")
fields array No Fields to include in response

Filter Object Schema

Field Type Description
column string Database column name to filter on
operator string Operator: =, !=, >, <, >=, <=, LIKE, IN
value any Value to compare against

PowerShell Example - Complex Filtering

# Advanced project filtering
$body = @{
    filters = @(
        @{
            column = "current_status"
            operator = "="
            value = "In Progress"
        },
        @{
            column = "strategic_portfolio"
            operator = "="
            value = "Technology"
        },
        @{
            column = "revenue_est_current_year"
            operator = ">"
            value = 50000
        }
    )
    logical_operator = "AND"
    fields = @(
        "project_name",
        "current_status", 
        "strategic_portfolio",
        "revenue_est_current_year",
        "start_date_est",
        "end_date_est",
        "resource_hours_planned",
        "resource_cost_planned"
    )
} | ConvertTo-Json -Depth 3

$response = Invoke-WebRequest -Uri "http://localhost:5000/projects/dynamic_filter" -Method POST -Body $body -ContentType "application/json"
$projects = $response.Content | ConvertFrom-Json

Write-Host "High-Value In-Progress Technology Projects:"
Write-Host "="*60

foreach ($project in $projects) {
    Write-Host "Project: $($project.project_name)"
    Write-Host "  Status: $($project.current_status)"
    Write-Host "  Revenue: `$$($project.revenue_est_current_year)"
    Write-Host "  Timeline: $($project.start_date_est) to $($project.end_date_est)"
    Write-Host "  Resources: $($project.project_resource_hours_planned) hours, `$$($project.project_resource_cost_planned)"
    Write-Host ""
}

PowerShell Example - Get All Fields

# Get all available fields for projects
$body = @{
    filters = @()
    fields = @("all")  # Special keyword to get all available fields
} | ConvertTo-Json -Depth 3

$response = Invoke-WebRequest -Uri "http://localhost:5000/projects/dynamic_filter" -Method POST -Body $body -ContentType "application/json"
$projects = $response.Content | ConvertFrom-Json

# Show first project with all fields
if ($projects.Count -gt 0) {
    Write-Host "Available Fields in Projects API:"
    $projects[0].PSObject.Properties | ForEach-Object {
        Write-Host "  $($_.Name): $($_.TypeNameOfValue)"
    }
}
